# Sunny Tools (WallSplitter)

Revit용 실무 편의 애드인 모음입니다. Revit 리본에 전용 탭 **"Sunny Tools"**를 추가하며, **Revit 2023 ~ 2027**을 하나의 소스로 지원합니다.

## 주요 기능

### 벽체 분리 / 바닥 분리
복합 구조(여러 레이어)로 이루어진 벽체·바닥을 레이어별 단일 재질 벽체·바닥으로 분리합니다.
- 선택한 벽/바닥의 각 레이어(멤브레인 제외)를 별도의 요소로 생성하고, 원본은 삭제합니다.
- 직선/호(Arc) 형태의 벽 위치선, 편집된 프로파일(스케치)도 그대로 복사됩니다.
- 분리 후 같은 재질끼리는 자동으로 코너 트림 및 재결합됩니다.
- 유형 이름은 두 가지 방식으로 지정할 수 있습니다.
  - **템플릿 방식**: 프로젝트명/건물명/커스텀 텍스트 + 재질명/두께를 조합한 규칙으로 자동 생성
  - **직접 지정 방식**: 레이어마다 기존 유형을 직접 검색/선택 (선택 내용은 "단일/복수" 설정에 따라 세션 동안 재사용 가능)

### NAMER (이름 일괄 변경)
뷰/시트/패밀리/유형/재료 이름을 필터링 후 일괄 변경하는 도구입니다.
- 포함/불포함/일치/불일치/시작함/끝남 등 다양한 필터 모드
- 문자열 치환, 위치 삽입, 구분자 기준 자리바꾸기, 구간 삭제 등 4가지 변경 방식
- 적용 → 최종 적용의 2단계 구조로, 여러 작업을 이어서 누적 적용 후 한 번에 모델에 반영

### 재료 지정 (Material Assign)
여러 패밀리/유형에 재료를 일괄 지정, 재료 삭제(사용처 자동 재지정), 재료의 클래스/설명 일괄 변경을 지원합니다.

## 지원 Revit 버전

| Revit 버전 | 대상 프레임워크 |
|---|---|
| 2023 | net48 |
| 2024 | net48 |
| 2025 | net8.0-windows |
| 2026 | net8.0-windows |
| 2027 | net10.0-windows |

## 설치 (사용자용)

1. `SunnyTools_Installer.zip`을 내려받아 압축을 풉니다.
2. `SunnyTools_Install.exe`를 실행합니다. 설치된 Revit 버전을 자동으로 감지해 해당 버전에 맞는 파일과 `.addin` 매니페스트를 설치합니다.
3. Revit을 실행하면 리본에 "Sunny Tools" 탭이 나타납니다.

## 개발 / 빌드

.NET SDK가 설치되어 있어야 합니다.

```powershell
# 애드인만 특정 연도로 빌드
dotnet build WallSplitter/WallSplitter.csproj -c Release2026

# 설치 프로그램 빌드 (5개 연도 전체를 자동으로 빌드해 포함)
dotnet publish SunnyToolsInstaller/SunnyToolsInstaller.csproj -c Release
```

같은 소스를 `Debug2023`~`Release2027` 10개 Configuration으로 빌드하며, 각 Configuration의 4자리 연도가 `$(RevitVersion)`으로 읽혀 대상 프레임워크와 `Nice3point.Revit.Api.RevitAPI`/`RevitAPIUI` 패키지 버전을 결정합니다.

## 프로젝트 구조

- `WallSplitter/` — 애드인 본체 (벽체 분리, 바닥 분리, NAMER, 재료 지정, 설정 UI)
- `SunnyToolsInstaller/` — 5개 연도 빌드를 감지·설치하는 독립 설치 프로그램
- `WallSplitter.slnx` — 솔루션 파일
- `Old_Versions/` — 과거 설치 프로그램 빌드 산출물 보관용 (최신 산출물은 루트에 별도 유지)

## 알려진 제한사항

- 원본 벽/바닥에 호스팅된 문/창은 재호스팅되지 않고 원본과 함께 삭제됩니다.
- 코너 자동 트림은 직선(Line) 기반 벽 쌍만 지원하며, 호(Arc) 코너는 미지원입니다.
- 두께는 mm 단위 반올림 표시되어, 소수점 차이가 있는 레이어는 같은 유형으로 병합될 수 있습니다.
- 재료/유형 사용처 재지정은 `ElementType` 단위까지만 스캔하며, 인스턴스별 재료 오버라이드는 대상이 아닙니다.

더 자세한 설계 배경과 이력은 `CLAUDE.md`를 참고하세요.
